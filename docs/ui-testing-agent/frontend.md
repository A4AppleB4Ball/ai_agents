# Frontend

This document covers the frontend architecture for the UI Testing Agent, including the split panel layout, browser viewport rendering, WebSocket communication, state management, and component structure.

## Split Panel Layout

**File:** `web/src/features/ui-agent/components/ui-agent-layout.tsx`

The UIAgentLayout renders a horizontally split panel with:

| Property | Value |
|----------|-------|
| Default split | 40% chat / 60% browser |
| Minimum | 30% for either pane |
| Maximum | 70% for either pane |
| Animation | Spring physics (stiffness: 300, damping: 30) |
| Drag cursor | `col-resize` |

The drag handle renders a 6px-wide touch target with:
- A 2px visible track line (colored with `--mulberry` when dragging, `--border` otherwise)
- A centered grip indicator (visible on hover/drag) with three dots

The layout connects to the browser WebSocket on mount by:
1. Building the browser WS URL from the chat WS URL (replacing `/chat/ws` with `/browser/ws`)
2. Passing the URL and current `sessionKey` to the `useBrowserWebSocket` hook

## Browser Panel Modes

**File:** `web/src/features/ui-agent/components/browser-panel.tsx`

### Live Mode

Active when browser sessions exist. Renders:
- `SessionTabs` - horizontal tab bar for switching between parallel sessions
- `LiveViewport` - screencast frame display with action overlay

When no sessions are active, shows an empty state with a pulsing Globe icon and guidance text.

### Report Mode

Displays the latest test report generated by `finalize_run`. The report HTML is fetched from:
```
GET /agent/v1/ui-testing/runs/{run_id}/report
```

### History Mode

Lists past test runs fetched from:
```
GET /agent/v1/ui-testing/runs
```

Returns run summaries (run_id, summary, generated_at) displayed in reverse chronological order.

## LiveViewport

**File:** `web/src/features/ui-agent/components/live-viewport.tsx`

### Frame Rendering

The viewport renders CDP screencast frames as JPEG images:

1. `useBrowserStore` provides the latest `currentFrame[activeSessionId]` (base64 string)
2. `useScreencast` hook throttles rendering via `requestAnimationFrame`:
   - Stores the latest frame in a ref
   - On each animation frame, updates `displaySrc` state
   - Returns `data:image/jpeg;base64,{data}` or `null`
3. The `<img>` element uses `object-contain` to fit within the viewport
4. Frame transitions use framer-motion with 150ms opacity fade

### Empty State

When no frame data exists (session not yet streaming), shows:
- Pulsing `Monitor` icon (mulberry color, 2.5s animation cycle)
- "Waiting for browser session..." text
- Animated scan line effect (linear 3s from top to bottom, repeating)

### Loading State

Detected when the latest action is `page_load` or `navigate`:
- Passed to `BrowserChrome` to show a loading indicator in the URL bar

## ActionOverlay

**File:** `web/src/features/ui-agent/components/action-overlay.tsx`

Bottom overlay that displays the current browser action:

| Action Type | Color | Icon |
|-------------|-------|------|
| `click` | #830051 (mulberry) | circle-dot |

| `page_load` | #d97706 (amber) | reload |
| `scroll` | #7c3aed (purple) | arrows-vertical |

Layout: `[icon] [TYPE badge] [description text] [selector badge]`

Animation:
- Enter: opacity 0 -> 1, y 8px -> 0px (200ms)
- Exit: opacity 1 -> 0, y 0px -> 4px (200ms)
- Uses `AnimatePresence` with `mode="wait"` for smooth transitions

Auto-clears after 2 seconds of inactivity (managed by the WebSocket hook).

## SessionTabs

**File:** `web/src/features/ui-agent/components/session-tabs.tsx`

Horizontal tab bar for parallel browser sessions. Only renders when more than one session exists.

Each tab displays:
- Status dot (green if screencast active, gray if inactive)
- Session name (truncated to 120px max width)
- Active underline (mulberry color, spring animation with `layoutId`)

Clicking a tab calls `setActiveSession(session_id)` which:
- Updates `activeSessionId` in the store
- LiveViewport automatically renders frames for the new active session
- No WebSocket re-subscription needed (frames for all subscribed sessions are received)

## Browser WebSocket

**File:** `web/src/features/ui-agent/hooks/use-browser-websocket.ts`

### Connection

The hook connects to `ws://host:port/agent/v1/browser/ws` (or `wss://` in production).

Authentication: Token passed as query parameter `?token=...`

### Auto-Reconnect

Implements exponential backoff reconnection:

| Attempt | Delay |
|---------|-------|
| 1 | 1s |
| 2 | 2s |
| 3 | 4s |
| 4 | 8s |
| 5 | 16s |
| 6+ | 30s (max) |

Resets to 0 on successful connection.

### Subscription Model

**Client -> Server messages:**

```typescript
// Subscribe to receive frames/actions for specific sessions
{ "type": "subscribe", "session_ids": ["session-abc", "session-def"] }

// Stop receiving frames/actions for specific sessions
{ "type": "unsubscribe", "session_ids": ["session-abc"] }
```

**Server -> Client messages:**

```typescript
// Screencast frame (base64 JPEG)
{
  "type": "frame",
  "session_id": "session-abc",
  "data": "/9j/4AAQSkZ...",  // base64 JPEG
  "timestamp": 1717500000.123
}

// Browser action event
{
  "type": "action",
  "session_id": "session-abc",
  "action": "click",
  "detail": { "selector": "#btn", "description": "Submit" },
  "timestamp": 1717500001.456
}

// New session started (auto-subscribe)
{
  "type": "session_started",
  "session_id": "session-abc",
  "session_name": "google-search",
  "url": "https://www.google.com",
  "created_at": 1717500000.0,
  "screencast_active": true
}

// Session closed
{
  "type": "session_closed",
  "session_id": "session-abc"
}

// Subscription acknowledgements
{ "type": "subscribed", "session_ids": ["session-abc"] }
{ "type": "unsubscribed", "session_ids": ["session-abc"] }

// Errors
{ "type": "error", "message": "Unknown message type: ..." }
```

### Chat-Aware Session Switching

When `chatSessionKey` changes (user switches chat conversations):

1. Unsubscribe from all currently subscribed session IDs
2. Reset Zustand store state (sessions, frames, actions)
3. Fetch active sessions for the new chat via REST:
   ```
   GET /agent/v1/ui-testing/chat-sessions/{chatSessionKey}
   ```
4. Add fetched sessions to the store
5. Subscribe to the new session IDs

This ensures the browser panel always shows sessions belonging to the current chat.

### Message Handling

| Message Type | Handler |
|-------------|---------|
| `frame` | `browserStore.setFrame(sessionId, data)` |
| `action` | `browserStore.setAction(sessionId, action)` + auto-clear timer (2s) |
| `session_started` | `browserStore.addSession(session)` + auto-subscribe |
| `session_closed` | `browserStore.removeSession(sessionId)` |

## Zustand Store Structure

**File:** `web/src/features/ui-agent/store/browser-store.ts`

```typescript
interface BrowserStoreState {
  // Session tracking
  sessions: BrowserSessionInfo[];         // All active sessions for current chat
  activeSessionId: string | null;          // Session currently displayed in viewport

  // Streaming data (keyed by session_id)
  currentFrame: Record<string, string>;    // Latest base64 frame per session
  currentAction: Record<string, BrowserAction | null>;  // Latest action per session

  // Panel state
  panelMode: PanelMode;                    // "live" | "report" | "history"
  currentReport: TestReport | null;        // Report being displayed
  runs: TestRunSummary[];                  // Historical run list

  // Connection state
  isConnected: boolean;                    // Browser WebSocket connected
  chatSessionKey: string | null;           // Current chat association

  // Actions
  setFrame(sessionId: string, data: string): void;
  setAction(sessionId: string, action: BrowserAction | null): void;
  addSession(session: BrowserSessionInfo): void;
  removeSession(sessionId: string): void;
  setActiveSession(sessionId: string | null): void;
  setPanelMode(mode: PanelMode): void;
  setReport(report: TestReport | null): void;
  setRuns(runs: TestRunSummary[]): void;
  setConnected(connected: boolean): void;
  setChatSessionKey(key: string | null): void;
  reset(): void;
}
```

### Key Behaviors

- `addSession`: Appends to sessions array; sets as active if no active session exists
- `removeSession`: Removes session, cleans up frame/action data, falls back active session
- `setChatSessionKey`: Resets all session-related state (sessions, frames, actions, activeSessionId)
- `reset`: Returns the entire store to initial state

## Type Definitions

**File:** `web/src/features/ui-agent/types/browser.ts`

```typescript
type BrowserActionType = "click" | "navigate" | "input" | "page_load" | "scroll" | "hover";

interface BrowserFrame {
  session_id: string;
  data: string;        // base64 JPEG
  timestamp: number;
}

interface BrowserAction {
  session_id: string;
  action: BrowserActionType;
  detail: {
    url?: string;
    x?: number;
    y?: number;
    text?: string;
    selector?: string;
    description?: string;
  };
  timestamp: number;
}

interface BrowserSessionInfo {
  session_id: string;
  session_name: string;
  url: string;
  created_at: number;
  screencast_active: boolean;
}

type PanelMode = "live" | "report" | "history";

interface TestRunSummary {
  run_id: string;
  target_url: string;
  total: number;
  pass: number;
  fail: number;
  skip: number;
  duration_ms: number;
  created_at: string;
}

interface TestCaseResult {
  id: string;
  name: string;
  status: "pass" | "fail" | "skip";
  steps: TestStepResult[];
  duration_ms: number;
  error?: string;
}

interface TestStepResult {
  step: number;
  description: string;
  status: "pass" | "fail";
  screenshot_url?: string;
  timestamp: number;
}

interface TestReport {
  run_id: string;
  summary: { total: number; pass: number; fail: number; skip: number };
  test_cases: TestCaseResult[];
  html?: string;
  generated_at: string;
}

type BrowserWsMessage =
  | ({ type: "frame" } & BrowserFrame)
  | ({ type: "action" } & BrowserAction)
  | ({ type: "session_started" } & BrowserSessionInfo)
  | ({ type: "session_closed" } & { session_id: string });
```

## Customization

### Changing the Split Ratio

In `ui-agent-layout.tsx`, modify the constants:

```typescript
const MIN_SPLIT = 0.3;    // Minimum pane width (30%)
const MAX_SPLIT = 0.7;    // Maximum pane width (70%)
const DEFAULT_SPLIT = 0.4; // Default chat width (40%)
```

### Adding Panel Modes

In `browser-panel.tsx`, extend the `MODE_TABS` array:

```typescript
const MODE_TABS: { id: PanelMode; label: string; icon: typeof Radio }[] = [
  { id: "live", label: "Live", icon: Radio },
  { id: "report", label: "Report", icon: FileText },
  { id: "history", label: "History", icon: FolderClock },
  // Add new mode here
];
```

Also update the `PanelMode` type in `types/browser.ts`.

### Customizing Action Colors

In `action-overlay.tsx`, modify the `ACTION_COLORS` and `ACTION_ICONS` maps:

```typescript
const ACTION_COLORS: Record<BrowserActionType, string> = {
  click: "#830051",
  navigate: "#059669",
  input: "#2563eb",
  page_load: "#d97706",
  scroll: "#7c3aed",
  hover: "#0891b2",
};
```

### Adjusting Frame Rate Rendering

The `useScreencast` hook uses `requestAnimationFrame` for throttling. To reduce CPU usage on low-powered devices, you could add frame skipping logic in the hook.
