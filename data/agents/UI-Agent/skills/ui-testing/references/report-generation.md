# Test Report Generation

## Report Format

The final report is a self-contained HTML document with embedded screenshots (base64
data URIs). It requires no external dependencies to render.

## Report Structure

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>UI Test Report - {application_name} - {date}</title>
  <style>/* embedded styles */</style>
</head>
<body>
  <header>
    <h1>UI Test Report</h1>
    <div class="meta">
      <p>Application: {application_url}</p>
      <p>Date: {iso_timestamp}</p>
      <p>Browser: Chromium (Playwright)</p>
      <p>Tester: UI-Agent (Automated)</p>
      <p>Environment: {environment}</p>
    </div>
  </header>

  <section class="summary">
    <h2>Executive Summary</h2>
    <div class="stats">
      <div class="stat total">Total: {total}</div>
      <div class="stat pass">Passed: {pass}</div>
      <div class="stat fail">Failed: {fail}</div>
      <div class="stat skip">Skipped: {skip}</div>
    </div>
    <div class="pass-rate">Pass Rate: {pass_rate}%</div>
    <div class="duration">Total Duration: {total_duration_ms}ms</div>
  </section>

  <section class="test-cases">
    <h2>Test Cases</h2>
    <!-- One block per test case -->
    <article class="test-case {status}">
      <h3>{case_name} <span class="badge {status}">{STATUS}</span></h3>
      <p class="description">{case_description}</p>
      <p class="timing">Duration: {duration_ms}ms</p>

      <div class="steps">
        <ol>
          <li class="step {step_status}">
            <span class="step-desc">{step_description}</span>
            <span class="step-status">{step_status}</span>
            <img src="data:image/png;base64,{screenshot}" alt="{step_description}">
          </li>
          <!-- more steps -->
        </ol>
      </div>

      <!-- If failed -->
      <div class="failure-analysis">
        <h4>Failure Analysis</h4>
        <p>{error_description}</p>
        <pre>{console_errors}</pre>
      </div>
    </article>
  </section>

  <footer>
    <p>Generated by UI-Agent | {iso_timestamp}</p>
  </footer>
</body>
</html>
```

## Generation Rules

1. Screenshots MUST be embedded as base64 data URIs — no external file references
2. The HTML must be fully self-contained (no CDN links, no external CSS/JS)
3. Use the company brand color `#830051` (mulberry) for headers and accents
4. Include timing data for each case and total run
5. Failed cases must include a "Failure Analysis" section with error description
6. The report must be printable (use `@media print` rules)
7. Include environment metadata (URL, browser, date, tester identity)

## Generating via Script

For complex reports, use `scripts/generate_report.py`:

```bash
python scripts/generate_report.py --input test-events.json --output report.html
```

The script reads a JSON file containing all TEST_EVENT payloads and produces the HTML.

For simpler runs, you may generate the HTML inline in your response.

## Test Case Naming Convention

Follow enterprise naming standards:
- ID format: `tc-NNN` (zero-padded 3 digits)
- Name: Short imperative phrase ("Verify login flow", "Validate form submission")
- Description: One sentence explaining what is validated and the expected outcome
